# Google Engineering Practices Documentation

구글에는 모든 언어(language) 그리고 모든 프로젝트를 위한 많은 일반화된 엔지니어링 관행이 있다. 
이 곳에 우리가 이때까지 만들어온 좋은 관행들을 모아두었다. 
이 문서들은 다양한 오픈소스 프로젝트나 다양한 기관이나 기업에서 활용하면 도움이 될 수 있다. 
그래서 우리는 이 내용들을 공개하여 누구나 사용할 수 있게 하고자 한다.

현재, 아래의 내용들이 포함되어 있다.

*   [구글 코드리뷰 가이드(Google's Code Review Guidelines)](review/index.md), 아래의 두 항목으로 구분되어 있다.:
    *   [코드 리뷰어 가이드(The Code Reviewer's Guide)](review/reviewer/index.md)
    *   [코드 리뷰이? 가이드(The Change Author's Guide)](review/developer/index.md)

## 용어(Terminology)

본 문서의 정보들을 읽기 위해서 필요한, 구글 내부에서 사용하는 기술 용어들이다.:

*   **CL**: 변경 내용 목록 / 변경 목록 / changelist, 개발한 변경사항이나, 코드 리뷰를 진행할 목록 또는 version control에 제출된 내용들을 의미하며 보통 change나 patch 또는 pull-request로 표헌을 많이 한다.
*   **LGTM**: "내가 봤을땐 좋아." 를 의미하며, 변경된 내용에 동의 한다는 의미로 사용된다.
